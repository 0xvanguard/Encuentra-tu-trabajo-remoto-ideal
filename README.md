# Encuentra tu trabajo remoto ideal

Estrategia operativa de empleabilidad remota internacional para perfil **Python + Automation + Cloud + Security aplicada** desde Colombia.

No es teoría. Es material listo para ejecutar: CV, LinkedIn, READMEs vendibles, listas de empresas, mensajes de outreach y un plan de saneo del propio GitHub.

## Punto de entrada conceptual

> **[`ESTRATEGIA.md`](ESTRATEGIA.md)** — documento maestro. El porqué detrás de cada entregable: diagnóstico, ventajas, riesgos, segmentación de mercados, rankings, plan 14/30/90, KPIs y errores a evitar. Léelo una vez completo antes de ejecutar los `docs/`.

## Cómo usar este repo

Lee los documentos en orden. Cada uno se puede usar de forma independiente, pero el orden refleja el flujo natural de ejecución:

1. **Decidir cuánto cobrar** → `docs/00-rango-salarial.md`
2. **Tener el CV listo** → `docs/01-cv-backend-automation.md`
3. **Renovar LinkedIn** → `docs/02-linkedin-headline-about.md`
4. **Empaquetar 3 repos vendibles** → `docs/03-readmes-premium/`
5. **Saber a quién aplicar** → `docs/04-empresas-y-plataformas.md`
6. **Enviar mensajes que conviertan** → `docs/05-mensajes-outreach.md`
7. **Limpiar el GitHub para no autosabotearte** → `docs/06-saneo-github.md`

## Reglas de ejecución (no negociables)

- **Tres frentes en paralelo desde el día 1:** aplicar a empleo + mejorar activos visibles + abrir canal freelance. No esperar a que todo esté perfecto.
- **Doble rango salarial:** piso de supervivencia (lo mínimo para no morir económicamente) y rango objetivo (lo que vale tu perfil con activos pulidos). Negociar siempre desde el rango objetivo, aceptar el piso solo si la urgencia lo exige.
- **Aplicar selectivo, no masivo:** 8–12 aplicaciones diarias bien personalizadas valen más que 50 con copy-paste.
- **Medir cada semana:** si después de 30 días la tasa de respuesta es <3%, el problema no es volumen, son los activos.

Núcleo: Colombia (USD), México (fintech), Argentina (fintech con USD), Chile (Get on Board).
