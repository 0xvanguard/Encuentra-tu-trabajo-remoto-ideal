# Doc 00 — Rango salarial doble: supervivencia vs objetivo

## Por qué un solo número te hace daño

Si entras a una negociación con un solo número en la cabeza, lo vas a soltar mal:
- Si el número es bajo, te van a pagar el bajo y te quedas atrapado.
- Si el número es alto y tienes urgencia económica real, vas a quebrarte mentalmente al primer "no".

La salida es manejar **dos rangos** en paralelo y saber cuándo usar cada uno.

## Los dos rangos para tu perfil

### A. Piso de supervivencia
**800–1.000 USD/mes** (≈ 3.2M – 4M COP)

- Es lo que aceptarías **solo si la urgencia es 30 días o menos** y la oferta es de un cliente con quien no vas a quedarte más de 3 meses.
- Por encima del 2.8M COP que tenías como expectativa, porque ese número estaba subvaluándote.
- Útil para: primer cliente Workana de bajo riesgo, contrato puente, freelance corto plazo, BPO LATAM.
- Regla: **nunca lo dices primero**. Solo aparece si ya negociaste y cerraste todo lo de arriba.

### B. Rango objetivo (tu suelo real de negociación)
**1.500–2.500 USD/mes** (≈ 6M – 10M COP)

- Es el rango con el que abres conversaciones de empleo formal o freelance B2B.
- Justificado por tu stack: Python + Docker + Firebase/GCP + automation + security aplicada + portafolio público.
- Útil para: fintechs LATAM, startups producto, empresas argentinas/chilenas pagando USD, nearshore USA tier 2.
- Regla: si no llegan al piso de este rango (1.500), o subes scope o pasas a la siguiente vacante.

### C. Rango aspiracional (el que persigues a 6–12 meses)
**3.000–5.000 USD/mes** (≈ 12M – 20M COP)

- Esto es nearshore USA tier 1 (BairesDev, Encora, FullStack Labs, Capitole) + empresas argentinas premium (dLocal, Pomelo) + startups USA Series A–B remote-LATAM-friendly.
- Requiere: inglés C1 funcional, 6+ meses de track record remoto verificable, GitHub saneado, 1–2 referencias internacionales.
- Lo mencionas solo cuando aplicas a esos targets, no como ancla generalizada.

## Cómo se usa en la práctica

| Situación | Qué decir |
|-----------|-----------|
| Primera aplicación, no preguntan salario | No mencionar nada. |
| Primer mensaje pregunta "expectativa salarial" | "Mi rango se mueve entre **1.500 y 2.500 USD mensuales** dependiendo del scope, modalidad de contratación y duración. ¿Cuál es el rango presupuestado para esta posición?" |
| Te ofrecen menos de 1.500 USD para empleo formal | "Para un rol con esta combinación de stack, mi piso para empleo formal es 1.500. Si el rango aprobado está por debajo, ¿hay flexibilidad por ajuste de scope, bonos o equity?" |
| Te ofrecen 1.000 USD pero la urgencia económica es real y es freelance corto | Aceptar como **contrato puente** explícito de 2–3 meses, no como salario fijo. Mientras tanto, seguir aplicando arriba. |
| Vacante de USA-direct o nearshore tier 1 | Anclar en 3.000–4.000 USD y negociar desde ahí. |

## Cómo justificar el salto sobre tu expectativa actual

Tenías 2.8M COP (~700 USD) como mente puesta. Subir a 1.500 no es codicia, es realismo de mercado. Lo justificas así (interno, para ti):

- Un junior dev Python con Docker en Bogotá presencial cobra ya 4–6M COP en 2026.
- Cualquier vacante remoto **internacional** debe pagar mínimo 1.5x ese número porque no hay carga prestacional, no hay oficina, y compites por talento global.
- Tu stack cruzado (security + automation + cloud) suma 20–35% sobre el dev Python plano.
- Por debajo de 1.500 USD remoto internacional, el cliente está extractivo y te frena el track record.

## Señales de que tienes que subir el rango

- Cierras 2 contratos seguidos sin negociar → estás dejando dinero en la mesa.
- En las primeras 3 entrevistas la respuesta a tu cifra fue "está dentro de presupuesto" → estás abajo del mercado.
- Recibes 3 ofertas en una semana → el mercado te quiere más de lo que pides.

## Señales de que tienes que bajar el rango temporalmente

- 60+ aplicaciones serias, 0 entrevistas avanzadas → o el activo está flojo, o el rango no encaja con tu perfil real *hoy*. Revisar primero el activo.
- Runway financiero <30 días → tomar contrato puente al piso de supervivencia mientras sigues buscando arriba. **No detener la búsqueda principal.**

## Lectura honesta

El error más caro que cometen los devs juniors LATAM no es pedir mucho. Es pedir poco, conseguirlo rápido, y quedarse atrapados ahí 18 meses porque cambiar trabajo cuesta.

Subir 30% de salario una vez dentro es 5x más difícil que negociarlo bien antes de firmar.
