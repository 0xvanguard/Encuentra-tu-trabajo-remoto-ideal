# Doc 02 — LinkedIn: titular, About y configuración mínima

LinkedIn es la herramienta de alta conversión cuando el perfil está bien armado. Aquí va lo exacto a copiar e iterar.

## Configuración mínima antes de pulir el contenido

| Elemento | Acción |
|---|---|
| URL personalizada | `linkedin.com/in/tunombre-apellido` (cámbiala desde "Editar perfil público y URL"). |
| Foto de perfil | Foto clara, fondo neutro, vestimenta de oficina informal. Sin selfies, sin filtros, sin sombrero. |
| Banner | Imagen sobria con tu stack o cita corta. Plantillas gratis en Canva. Evita gradientes psicodélicos. |
| Ubicación | "Bogotá, Colombia" (o tu ciudad real). Activar `Open to remote work — Worldwide`. |
| Idioma del perfil | Crear el perfil en **inglés como principal** y agregar versión en español. Esto duplica visibilidad ante reclutadores USA/EU. |
| Industria | "Computer & Network Security" o "Information Technology & Services". |
| Sección "Open to work" | Activar privadamente (solo recruiters), no con el banner verde encima de la foto (queda desesperado). Tipos de rol: "Backend Developer", "Automation Engineer", "DevSecOps Engineer". |
| Contacto | Email profesional + GitHub link en sección "Contact info". |

## Titular (Headline) — máximo 220 caracteres

### Versión inglés (recomendada como principal)

```
Python Automation Engineer | Docker · Firebase · GCP · Applied Security
Building secure backends and internal tools — Open to remote LATAM/USA
```

### Versión español (alternativa, si tu mercado objetivo es 100% LATAM)

```
Python Automation Engineer | Docker · Firebase · GCP · Seguridad aplicada
Backends e internal tools seguros por defecto — Remoto LATAM y USA
```

**Por qué este titular funciona**
- Empieza con un rol claro, no con "estudiante de" o "apasionado por".
- Carga 4 keywords técnicas que los recruiters buscan literal.
- Termina con disponibilidad geográfica explícita.
- Cabe en móvil sin truncar.

**Por qué evitamos otras versiones que parecen tentadoras**
- *"Senior Cybersecurity Engineer & AI Innovator"* → suena inflado, te filtra negativamente con técnicos.
- *"Apasionado por la tecnología y aprendiz constante"* → señal de junior sin foco. Lo evitas.
- Listar 8 tecnologías → te ven como junior que apila keywords.

## Sección Featured (lo más importante visualmente)

Pin estos 3 elementos arriba, en este orden:

1. **Link al repo `Security-Checklist-Ecommerce`** — con thumbnail de portada (puedes generarlo en Canva con título grande "Security Checklist for E-commerce").
2. **Link al repo `VanguardOps`** — thumbnail "IT Support Automation Platform".
3. **Link al repo `Attack-Surface-Validation-Lab.`** — thumbnail "Attack Surface Validation Lab".

> Cuando un recruiter abre LinkedIn móvil, lo primero que ve es Featured. Los pin de abajo (artículos, posts) los ven 1/10 personas. Por eso reservamos los 3 slots para los tres repos vendibles.

## Sección Skills (orden importa)

Los primeros 3 que LinkedIn muestra grandes son los que el algoritmo usa para matching. Pon EN ESTE ORDEN:

1. Python
2. Docker
3. Cybersecurity (o "Application Security")

Después: Firebase, Google Cloud Platform, FastAPI, Linux, Bash, Automation, REST APIs, Git, CI/CD.

> No pongas más de 25 skills. Más de 25, LinkedIn los esconde y no aportan al algoritmo.

## Sección Experience

Reflejar exactamente la sección "Experiencia" del CV. Los bullets pueden ser un poco más narrativos en LinkedIn (no estás peleando con un ATS aquí, estás peleando con un humano que escanea).

## Recomendación de actividad

Una vez que el perfil esté pulido:
- **1 publicación a la semana** mínimo. Tema: algo que aprendiste, un mini case-study de uno de tus repos, una opinión técnica corta.
- **3 comentarios sustanciales/semana** en posts de fundadores, CTOs o reclutadores tech LATAM.
- **No** publiques motivacionales tipo "el éxito está en levantarse temprano". Eso espanta recruiters técnicos.

## KPI de LinkedIn

| Métrica | Baseline | Meta 30 días | Meta 90 días |
|---|---|---|---|
| Profile views | ~10/sem | 80/sem | 300+/sem |
| Search appearances | ~20/sem | 150/sem | 500+/sem |
| InMails recibidos | 0 | 1–2/mes | 4–8/mes |
| Conexiones tech relevantes | actual | +150 | +500 |

Si después de 30 días con todo pulido los números no se mueven, el problema es **frecuencia de publicación + comentarios estratégicos**, no el perfil.
